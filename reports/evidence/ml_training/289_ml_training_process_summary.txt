1. Role de ce document
----------------------

Ce document resume clairement tout le processus ML training RubyBets.

Il sert a eviter que le dossier reports/evidence/ml_training/ devienne illisible avec plusieurs centaines de fichiers de preuves.
L'objectif est de conserver une lecture simple pour le suivi RNCP, la soutenance et les prochains developpements.

Le principe retenu est le suivant :
- conserver les preuves detaillees dans une archive ZIP ;
- garder ce fichier TXT comme resume narratif du processus ;
- continuer a rappeler que le ML reste experimental ;
- ne jamais confondre ces experimentations avec le scoring explicable V1 officiel.

3. Positionnement general conserve pendant toute la phase ML
------------------------------------------------------------

RubyBets reste une application web d'aide a la decision football avant-match.

RubyBets :
- ne permet pas de parier ;
- n'est pas un bookmaker ;
- ne promet aucun resultat sportif ;
- aide uniquement a structurer une analyse a partir de donnees reelles.

Le scoring explicable V1 reste le socle officiel du produit.

Toutes les experimentations ML documentees ici restent separees :
- de l'API officielle /api/matches/{match_id}/predictions ;
- du frontend ;
- de PostgreSQL ;
- de ml.features, sauf lecture technique quand elle est explicitement indiquee ;
- du scoring explicable V1 ;
- de toute promesse de resultat sportif.

4. Phase 1 - Premiere baseline ML 1X2
-------------------------------------

Fichiers principaux :
- 01_dataset_summary.txt
- 02_training_execution_log.txt
- 03_model_comparison.csv
- 04_logistic_regression_report.txt
- 05_random_forest_report.txt
- 06_xgboost_report.txt
- 07_best_model_decision.txt

Objectif :
Construire une premiere baseline de classification supervisee pour predire le resultat 1X2 :
- HOME_WIN ;
- DRAW ;
- AWAY_WIN.

Dataset utilise :
- environ 44 957 matchs nettoyes ;
- ligues principales : D1, E0, F1, I1, SP1 ;
- saisons historiques de 2000_2001 a 2024_2025 selon les experiences ;
- split chronologique pour eviter une evaluation artificielle.

Premieres features :
- forme recente domicile/exterieur ;
- buts marques recemment ;
- buts encaisses recemment ;
- fenetres last_5 au depart.

Resultat de reference :
- modele sauvegarde initial : LogisticRegression_balanced ;
- accuracy officielle initiale : 0.4669 ;
- F1 macro officiel : 0.4266.

Lecture :
Cette premiere baseline etait utile comme point de depart, mais trop faible pour devenir une fonctionnalite produit.
Elle a surtout servi a mettre en place une chaine ML propre : chargement, entrainement, evaluation, sauvegarde et preuves.

18. Decision finale actuelle
----------------------------

La meilleure reference prudente reste :
- V17.0 multi-market controlled selector ;
- accuracy : 0.8382 ;
- coverage : 0.7651.

La variante retenue pour l'objectif utilisateur est :
- V17.8 user objective BTTS selector ;
- accuracy : 0.8000 ;
- coverage : 0.7946 ;
- selected rows : 4235 ;
- integration de 1X2 strict, double chance, OVER_1_5 et BTTS.

Pourquoi V17.8 est retenue malgre une accuracy plus basse que V17.0 :
- elle repond mieux a l'objectif utilisateur multi-marches ;
- elle augmente la couverture ;
- elle rend BTTS plus visible ;
- elle garde un niveau global professionnel ;
- elle reste clairement experimentale.

Pourquoi V17.8 n'est pas integree officiellement :
- BTTS reste le marche le plus fragile ;
- certains segments restent sous les niveaux de prudence souhaites ;
- le modele n'est pas encore branche sur les vrais matchs RubyBets ;
- le scoring explicable V1 reste le moteur officiel.

24. Conclusion
--------------

La phase ML training RubyBets a evolue progressivement :

1. d'une baseline 1X2 faible mais propre ;
2. vers des features plus riches ;
3. puis vers une logique selective avec abstention ;
4. puis vers une decision produit plus intelligente : double chance pour les matchs incertains ;
5. puis vers un selecteur multi-marches ;
6. enfin vers V17.8, qui integre BTTS de maniere experimentale et plus proche de l'objectif utilisateur.

La conclusion actuelle est claire :
- V17.0 reste la reference prudente ;
- V17.8 est la variante experimentale cible pour l'objectif utilisateur ;
- le scoring explicable V1 reste le moteur officiel ;
- aucune integration produit finale de V17.8 n'est encore realisee ;
- toutes les preuves doivent rester archivees et tracables.
